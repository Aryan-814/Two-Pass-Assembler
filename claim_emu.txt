Compilation instructions:
=========================

    g++ emu.cpp -o emu
    ./emu <filename.o>

    * You must provide a valid machine-code object file (.o) generated by the assembler.
    * Example: ./emu bubblesort.o
    * The object file should be present in the same folder as the cpp file.
    * Upon running, the emulator will enter an interactive command-line interface.
    * Available commands include:
        -run   : Run the entire program until HALT.
        -t     : Run a single instruction (trace mode) and show registers.
        -reg   : Show current state of A, B, SP, and PC registers.
        -dump  : Print a smart memory dump (skips empty blocks).
        -isa   : Show instruction set using
        -quit  : Exit the emulator.


Assumptions:
============
    * The input file is a strictly formatted 32-bit binary file generated by the SIMPLEX assembler.
    * The program execution always begins at PC = 0.
    * The Stack Pointer (SP) is not strictly initialized by the hardware; the assembly program is responsible for allocating its own stack space (e.g., using `ldc 0x1000`, `a2sp`).
    * The emulator operates in a 16MB virtual memory space (1 << 24 integers).


=============================
Program format and structure:
=============================
1. The Emulator (emu.cpp):
    * Architecture: Implements an accurate Fetch-Decode-Execute cycle. The Program Counter (PC) increments immediately after the instruction fetch to perfectly mimic hardware clock cycles and calculate branch offsets natively.
    * Sign Extension: Utilizes native C++ arithmetic right shifts (`inst >> 8`) on signed 32-bit integers to automatically and efficiently sign-extend negative 24-bit offsets (e.g., `adj -1`, backward branches).
    * Error Diagnostics & Errant Program Detection: The virtual CPU is highly defensive and will safely halt rather than crashing the host OS. It detects:
        - Memory Access Violations: Out-of-bounds pointer dereferencing in `ldl`, `stl`, `ldnl`, and `stnl` instructions.
        - Runaway Programs: PC incrementing out of the bounds of the loaded code.
        - Illegal Instructions: Catching undefined opcodes (e.g., trying to execute data).
    * Smart Memory Dump: The `-dump` function scans the entire 16MB memory space but only prints the active instruction memory and any modified memory blocks (like the call stack or dynamic arrays), ensuring clean and readable outputs.
    * Traceability: Includes built-in debugging traces for deep analysis of memory bottlenecks by tracking every read and write cycle.
