=============================
Program format and structure:
=============================
1. The Assembler (asm.cpp):
    * Architecture: Implements a "One Read" Two-Pass Assembler. Pass 1 parses and caches instructions internally into C++ memory (`std::map`) for Pass 2.
    * Completeness: Every instruction in the SIMPLEX ISA is implemented with accurate expected operand counts.
    * Advanced Directives: The `SET` instruction is fully implemented (for extra credit) to allow custom label value assignment.
    * PC-Relative Offsets: Accurately calculates branch offsets relative to `PC + 1` for branch instructions (`br`, `brz`, `brlz`, `call`).
    * Error Diagnostics: Robustly detects and logs:
        - Duplicate labels
        - Bogus label names (non-alphanumeric or starting with a number)
        - Unexpected or missing operands
        - Invalid number formats (e.g., malformed hex or string-integer mixes)
    * Code Quality: Features unused label detection. Labels that are defined but never referenced trigger a Warning in the `.log` file to help identify dead code.
    * Clean output generation: The listing file (.lst) perfectly aligns the PC, machine code (in 8-character hex), and original instruction for easy debugging.
